Q1 General Machine Learning: challenges, overfitting, and hyperparameter tuning pitfalls. ▼
General MLLecture 1–3
a)What are the main challenges of machine learning?
Answer

The 7 main challenges are: (1) Insufficient training data, (2) Non-representative data (sampling bias), (3) Poor-quality data (noise, outliers), (4) Irrelevant features (feature engineering needed), (5) Overfitting (model too complex), (6) Underfitting (model too simple), (7) No Free Lunch theorem — no single algorithm always wins.

Detail
  • Insufficient training data – most algorithms need thousands of examples to work well.
  • Non-representative training data – sampling bias means the training set doesn't reflect the real distribution.
  • Poor-quality data – outliers, errors, and noise cause the model to learn wrong patterns.
  • Irrelevant features – "garbage in, garbage out"; feature engineering is critical.
  • Overfitting – model memorises training data, fails on new instances (high variance).
  • Underfitting – model too simple to capture underlying structure (high bias).
  • No Free Lunch theorem – no single algorithm always wins; choice depends on the problem.
b)If a model performs perfectly on training data but poorly on new instances, what is this called and what are the solutions?
Answer

This is called overfitting — the model memorises training data rather than learning general patterns. Solutions: regularisation (Ridge/Lasso/Elastic Net), gather more data, reduce noise, simplify the model, use cross-validation, dropout (neural networks), or early stopping.

Detail
  • Regularisation (Ridge/Lasso/Elastic Net) — penalise large weights to constrain model complexity.
  • Gather more training data — more data reduces variance.
  • Reduce noise in the dataset — clean labels, remove outliers.
  • Simplify the model — fewer parameters, shallower tree, fewer layers.
  • Cross-validation — detects overfitting during training before test evaluation.
  • Dropout (neural networks) — randomly deactivate neurons during training.
  • Early stopping — halt training when validation loss starts increasing.
c)What can go wrong if you repeatedly tune hyperparameters to optimise performance on the test set?
Answer

This causes data leakage — the test set effectively becomes part of training. The model is tuned to the quirks of that specific test split, making reported performance over-optimistic. On truly unseen data, performance will be worse. Solution: use a separate validation set (or k-fold cross-validation) for tuning; keep the test set completely blinded.

Detail

The test set exists to simulate the real world. Once it is used to make any decision (including hyperparameter selection), it is no longer a fair estimate of generalisation. After many tuning iterations, the selected model is essentially over-fitted to the test set — but this cannot be detected from the reported score alone.

Q2 Performance measurements: regression metrics, confusion matrix, and AUC-ROC curve. ▼
Performance MetricsLecture 4
a)What metrics can measure the performance of a regression model? Explain what each measures and how to interpret it.
Answer

MAE = mean of |errors|, robust to outliers, easy to interpret. MSE = mean of squared errors, penalises large errors more. RMSE = √MSE, same units as the target variable, most commonly reported. R² = proportion of variance explained (1 = perfect, 0 = no better than predicting the mean, <0 = worse than mean). Lower MAE/MSE/RMSE is better; higher R² is better.

Detail
  • MAE = (1/n)·Σ|ŷᵢ − yᵢ|. Not affected by outlier magnitude. Easy to interpret as average error.
  • MSE = (1/n)·Σ(ŷᵢ − yᵢ)². Differentiable — useful for gradient-based training. Punishes large errors heavily.
  • RMSE = √MSE. Same units as target. Preferred for reporting since it's interpretable in context of target scale.
  • R² = 1 − SS_res/SS_tot. Tells you what fraction of variance the model explains. Negative R² means the model performs worse than a constant mean predictor.
b)What is a confusion matrix and how can it be used to evaluate a classification model? Include precision, recall, accuracy, and F1-score.
Answer

A confusion matrix shows TP, TN, FP, FN. From it: Accuracy = (TP+TN)/total (misleading for imbalanced data). Precision = TP/(TP+FP) — quality of positive predictions. Recall = TP/(TP+FN) — coverage of actual positives. F1 = 2·P·R/(P+R) — harmonic mean, balanced measure. Macro-F1 = average F1 per class with equal weight, best for imbalanced multi-class problems.

Detail

A confusion matrix is a table where rows = actual classes and columns = predicted classes. For a binary classifier:

  • TP: correctly predicted positive. TN: correctly predicted negative.
  • FP (Type I error): predicted positive, actually negative. FN (Type II error): predicted negative, actually positive.
  • Precision asks: "Are my positive predictions reliable?" — used when FP is costly (e.g., spam filter).
  • Recall asks: "Did I find all real positives?" — used when FN is costly (e.g., cancer screening).
  • F1-score balances both and is the metric of choice when classes are imbalanced.
c)How can you use an ROC curve to evaluate a classifier? What does AUC represent, and what shape does it have for a perfect classifier?
Answer

The ROC curve plots True Positive Rate (Recall) vs. False Positive Rate across all classification thresholds. AUC = 0.5 means random guessing (diagonal line); AUC = 1.0 means perfect discrimination. A perfect classifier has an L-shaped ROC curve — straight up from (0,0) to (0,1), then across to (1,1). AUC is threshold-invariant, making it ideal for comparing classifiers regardless of operating threshold.

Detail
  • TPR = TP/(TP+FN) (y-axis); FPR = FP/(FP+TN) (x-axis).
  • Each point on the curve corresponds to a different decision threshold (e.g., 0.3, 0.5, 0.7).
  • A model above the diagonal is better than random; below is worse than random.
  • AUC = probability that the classifier ranks a random positive example higher than a random negative one.
Q3 Gradient Descent variants, bias-variance tradeoff, and regularisation (Ridge, Lasso, Elastic Net). ▼
Gradient DescentLecture 3
a)Explain the Gradient Descent algorithm and its three main variants. How does each variant differ?
Answer

Gradient Descent minimises cost J(θ) by updating: θ = θ − α·∇J(θ). Batch GD: uses full dataset per step — stable but slow. Stochastic GD (SGD): uses 1 random sample per step — fast but noisy. Mini-batch GD: uses a small subset (e.g., 32–256 samples) — best balance of speed and stability; can use GPU parallelism; most widely used in deep learning.

Detail
  • Batch GD: guaranteed convergence to global minimum for convex functions. Impractical for millions of samples.
  • SGD: noisy updates help escape local minima. However, the cost function bounces around; a learning rate schedule is needed.
  • Mini-batch GD: exploits GPU matrix operations on batches, smooths out SGD noise, and is fast enough for large datasets. Standard in modern deep learning frameworks.
b)Explain the bias-variance tradeoff. What happens to the model's error when bias is too high or variance is too high?
Answer

Bias = error from oversimplified model (wrong assumptions) → underfitting: high training AND test error. Variance = sensitivity to training data fluctuations → overfitting: low training error, high test error. Total error = Bias² + Variance + irreducible noise. Simple models (linear): high bias, low variance. Complex models (deep trees): low bias, high variance. Goal: minimise total error on unseen data.

Detail
  • High bias: model misses the true signal in both train and test. Adding more data won't fix it — the model is too simple.
  • High variance: model memorises training data; small changes in data → large changes in model. Adding more data helps reduce variance.
  • The tradeoff: as you increase model complexity (reduce bias), variance increases; as you simplify (reduce variance), bias increases.
  • Regularisation, ensembling (bagging), and cross-validation all help manage this tradeoff.
c)Compare Ridge, Lasso, and Elastic Net regularisation. When would you use each?
Answer

Ridge (L2): penalty = λΣwᵢ² — shrinks all weights toward zero, none exactly zero; handles multicollinearity. Use when all features may matter. Lasso (L1): penalty = λΣ|wᵢ| — drives some weights to exactly zero → automatic feature selection. Use when only a few features truly matter. Elastic Net: L1+L2 combined — feature selection + weight shrinkage. Use when features are correlated or Lasso is too aggressive.

Detail
  • Ridge never zeroes out features — all inputs contribute with shrunk coefficients. Good default when you have many features with small but real effects.
  • Lasso's geometry (diamond-shaped constraint) causes exact zeros at corners. Produces sparse, interpretable models.
  • Elastic Net: when correlated features exist, Lasso picks one arbitrarily and drops the rest. Elastic Net keeps related features together. Generally preferred over pure Lasso when p (features) > n (samples).
Q4 Ensemble Learning: bagging, boosting, base learner choice, and Random Forest. ▼
Ensemble LearningLecture 5
a)Explain bagging and how it works. Include bootstrap sampling and how predictions are combined.
Answer

Bagging (Bootstrap AGGregating, Breiman 1996): (1) Draw L bootstrap samples (sample N instances with replacement) — each keeps ~63.2% unique instances. (2) Train L independent base learners in parallel. (3) Combine: majority voting (classification) or averaging (regression). Works because diverse learners make different errors → errors cancel out → lower variance. Random Forest = bagging + random feature subset at each split.

Detail
  • Out-of-bag (OOB) instances (~36.8% not in each bootstrap sample) can be used as a free validation set.
  • Bagging is especially effective for unstable learners — algorithms where small data changes cause large model changes (e.g., deep decision trees).
  • Concordet's theorem: if each learner is better than random and learners are independent, the ensemble approaches perfect accuracy as L → ∞.
b)What is the difference between bagging and boosting? When would you prefer bagging over boosting?
Answer

Bagging: trains models in parallel independently, equal weights, reduces variance, robust to noise. Boosting: trains sequentially — each model corrects the previous one's errors by giving higher weight to misclassified samples, reduces bias, can overfit noisy data. Prefer bagging when: data is noisy, you need parallelisation, or robustness is the priority over squeezing maximum accuracy.

Detail

Training orderParallel (independent)Sequential (each corrects previous)
Primary goalReduce varianceReduce bias
Noise handlingRobustCan overfit noise
Popular algorithmsRandom ForestAdaBoost, XGBoost, Gradient Boosting
c)How does the choice of base learner affect the performance of bagging vs. boosting?
Answer

For bagging: use high-variance, low-bias learners (e.g., deep unpruned decision trees) — bagging reduces their variance. Shallow trees in bagging are underpowered. For boosting: use high-bias, weak learners (e.g., decision stumps / shallow trees) — boosting reduces their bias sequentially. Complex learners in boosting overfit quickly. Key principle: the aggregation method must target the base learner's weakness.

Detail
  • If you put shallow trees (low variance) into bagging, there is little variance to reduce — the ensemble won't improve much over a single tree.
  • If you put complex trees (low bias, high variance) into boosting, the sequential re-weighting will quickly overfit the noise in the data.
  • Homogeneous ensembles (same algorithm) are most common. Stacking uses heterogeneous weak learners (different algorithms) combined by a meta-model.
Q5 Dimensionality Reduction: PCA, LDA vs PCA, feature selection vs feature extraction. ▼
Dimensionality ReductionLecture 6
a)What is PCA? What role do eigenvectors and eigenvalues play in it?
Answer

PCA finds directions of maximum variance in data and projects it to a lower-dimensional space. Steps: centre data → compute covariance matrix Σ → find eigenvectors & eigenvalues of Σ → sort by descending eigenvalue → project onto top-k eigenvectors. Eigenvectors are the principal components (orthogonal directions of variance). Eigenvalues measure how much variance each component captures. Keep enough components to retain ~95% of total variance.

Detail
  • The first eigenvector (largest eigenvalue) points in the direction of greatest spread in the data.
  • Eigenvectors are orthogonal — principal components are uncorrelated.
  • Explained variance ratio = eigenvalue_k / Σ eigenvalues. Used to decide how many components to keep.
  • Example: Eigenfaces — representing a face as a weighted combination of "eigenface" images (eigenvectors of face covariance matrix).
  • Pitfall: PCA is sensitive to feature scale — normalise features before applying PCA.
b)What is the difference between PCA and LDA? In what scenarios would you prefer LDA?
Answer

PCA is unsupervised — maximises total data variance, ignores class labels, can choose directions that don't separate classes; up to min(N,D) components. LDA is supervised — maximises between-class/within-class variance ratio, explicitly uses class labels for separability; at most (C−1) components. Prefer LDA when classification is the goal. Example: Fisherfaces (LDA) outperform Eigenfaces (PCA) for face recognition under varying illumination.

Detail
FeatureEigenfaces (PCA)Fisherfaces (LDA)

Uses labelsNoYes
Robust to lightingPoorBetter
Max dimensionsUp to min(N,D)At most (classes − 1)
c)Explain the difference between feature selection and feature extraction. Give one example of each.
Answer

Feature selection: selects a subset of the original features — originals are unchanged, results remain interpretable. Example: forward/backward search selecting "salt" and "saturated fat" from 8 nutritional features. Feature extraction: transforms original features into a new lower-dimensional space. New features are combinations of originals. Example: PCA creating new components as linear combinations of all 8 nutritional features — not directly interpretable.

Detail
  • Feature selection methods: filter (correlation, mutual information), wrapper (forward/backward search, RFE), embedded (Lasso, Random Forest importance).
  • Feature extraction methods: linear (PCA, LDA), non-linear (autoencoders, t-SNE).
  • Feature selection preserves interpretability — preferred when domain knowledge is important.
  • Feature extraction often achieves better compression and noise reduction — preferred when raw features are redundant or noisy.
Q6 Artificial Neural Networks: perceptron, activation functions, and preventing overfitting. ▼
ANNsLecture 7
a)Describe the perceptron model. What are its limitations?
Answer

The perceptron (Rosenblatt, 1957) computes z = Σwᵢxᵢ + b then applies a step function (output = 1 if z ≥ 0, else 0). Limitations: (1) Only solves linearly separable problems — cannot learn XOR. (2) Step function is non-differentiable → gradient descent cannot be applied. (3) No hidden layers → cannot represent non-linear functions. Solution: Multi-Layer Perceptron (MLP) with differentiable activation functions and backpropagation.

Detail
  • Each input xᵢ is multiplied by weight wᵢ; the bias b shifts the decision boundary.
  • The perceptron learning rule adjusts weights only when a prediction is wrong.
  • Multiple perceptrons → MLP. Adding hidden layers with non-linear activations allows learning complex decision boundaries.
  • Backpropagation (chain rule) propagates error gradients through layers to update weights.
b)What is the role of activation functions in neural networks? Describe three activation functions and their properties.
Answer

Activation functions introduce non-linearity — without them, stacking linear layers = one linear transformation, so depth provides no benefit. Three key functions: Sigmoid: σ(x)=1/(1+e⁻ˣ), range [0,1] — probability output but suffers vanishing gradients. ReLU: max(0,x) — no saturation for positive inputs, default for hidden layers. Softmax: converts logits to class probabilities summing to 1 — used in multi-class output layer.

Detail
  • Sigmoid: derivative ≤ 0.25 everywhere → multiplied through many layers → vanishing gradient. Also not zero-centered.
  • Tanh: range [−1,1], zero-centered. Still vanishes in deep nets but better than sigmoid.
  • ReLU: f(x) = max(0,x). Gradient = 1 for x>0, so no saturation. Risk: "dying ReLU" if neurons always receive negative input (gradient = 0 permanently).
  • Leaky ReLU / ELU: small negative slope for x<0, fixes dying ReLU.
  • Softmax: eˣⁱ / Σeˣʲ — output interpreted as probability distribution over classes.
c)How can you prevent overfitting when training a neural network? Mention at least three techniques.
Answer

Dropout: randomly deactivates a fraction of neurons per training step (e.g., 50%) → forces redundant representations; not applied at inference. Early stopping: monitor validation loss; halt training when it starts increasing. L1/L2 regularisation: add penalty on weights to the loss function (L2 shrinks all; L1 sparsifies). Also: Batch normalisation and data augmentation (artificially expand training set).

Detail
  • Dropout: at each step, each neuron deactivated with probability p. Forces the network to not rely on specific neurons, learning robust representations. At test time, all neurons are active but outputs scaled by (1-p).
  • Early stopping: keep the model checkpoint from the epoch with lowest validation loss. Simple but very effective.
  • Batch normalisation: normalises inputs of each layer to zero mean, unit variance per mini-batch. Reduces internal covariate shift, stabilises training, has regularising effect.
  • Data augmentation: for images — flip, rotate, crop, colour jitter. Creates effectively more training data from existing samples.
Q7 Training Deep Neural Networks: vanishing/exploding gradients, learning rate scheduling, optimisers. ▼
Deep Learning TrainingLecture 8
a)What are vanishing and exploding gradients? How do they affect training, and what are the remedies?
Answer

Vanishing gradient: gradients shrink exponentially during backprop (sigmoid derivative ≤ 0.25 per layer) → lower layers receive ~0 gradient → stop learning. Exploding gradient: gradients grow exponentially → unstable updates, NaN values (common in RNNs). Remedies: use ReLU (gradient=1 for x>0), proper weight initialisation (He/Xavier), Batch Normalisation, gradient clipping (for exploding), LSTM/GRU gates (for RNNs), residual/skip connections.

Detail
  • Glorot & Bengio (2010) first formally explained the vanishing gradient problem — showed that with sigmoid + uniform weight init, variance grows through layers until saturating.
  • ReLU fix: derivative = 1 for x>0 → no saturation on positive side → gradients flow freely through many layers.
  • Batch Normalisation: normalises layer inputs so activation rarely enters the saturating tails of sigmoid/tanh.
  • Gradient clipping: if gradient norm > threshold, rescale to threshold. Does not change gradient direction, just magnitude.
  • Residual connections (ResNet): output = F(x) + x. The identity skip path provides a direct gradient highway, enabling training of 100+ layer networks.
b)Why is the learning rate a critical hyperparameter? Describe two learning rate scheduling strategies.
Answer

LR too high → cost function diverges or oscillates. LR too low → extremely slow convergence, may get trapped. Step decay: reduce LR by a factor (e.g., ×0.1) every N epochs — simple, allows fast initial progress then fine tuning. Cosine annealing: LR follows a cosine curve from max to near zero — smooth, no abrupt drops; often combined with warm restarts to escape local minima.

Detail
  • A good strategy: start with a relatively large LR for fast convergence, then decay it as the model approaches a minimum to avoid overshooting.
  • Warm restarts: periodically reset LR to its maximum, allowing the model to explore the loss landscape and escape shallow local minima.
  • Cyclical LR: oscillates between min and max LR every few epochs. Can speed up convergence and improve generalisation.
c)Compare two faster optimisers (e.g., Adam and RMSProp). How do they improve over vanilla gradient descent?
Answer

RMSProp: maintains exponential moving average of squared gradients v; adapts per-parameter LR: θ = θ − (α/√v+ε)·g. Prevents AdaGrad's monotonic LR decay. Good for RNNs. Adam: combines momentum (running mean of gradients m̂) + RMSProp (v̂): θ = θ − α·m̂/√(v̂+ε). Bias-corrected. Generally fastest convergence, default choice for deep learning. Both adapt the LR per parameter, outperforming fixed-LR gradient descent.

Detail
  • Momentum: accumulates a velocity in direction of persistent gradient. Accelerates in consistent directions, dampens oscillations.
  • Nesterov Accelerated Gradient: computes gradient at the "look-ahead" position → smoother convergence than vanilla momentum.
  • AdaGrad: accumulates sum of squared gradients — LR shrinks monotonically. Good for sparse features but LR eventually → 0.
  • RMSProp: uses exponential decay instead of sum in AdaGrad → avoids LR dying out. Recommended for RNNs (Hinton).
  • Adam: default for most tasks. Typical hyperparameters: β₁=0.9, β₂=0.999, ε=1e-8, α=0.001.
Q8 Convolutional Neural Networks: architecture, advantages over FC networks, and transfer learning. ▼
CNNsLecture 9
a)Describe the architecture of a typical CNN used for image classification. What does each layer do?
Answer

Input (raw pixels) → Conv layer (learnable filters slide across image, detect local patterns; produce feature maps via weight sharing) → ReLU (sets negatives to zero, non-linearity) → Max Pooling (takes max in local window, reduces spatial size, translation invariance) → repeat Conv+ReLU+Pool → Fully Connected (flattened features → class scores) → Softmax (class probabilities summing to 1).

Detail
  • Convolutional layer: filter (kernel) slides across image. At each position: element-wise multiply filter × image patch, sum → one value in feature map. Each filter detects one pattern type. Multiple filters → stack of feature maps.
  • Weight sharing: same filter applied across all positions → far fewer parameters than fully-connected. A 3×3 filter has only 9 weights regardless of image size.
  • ReLU: sets all negative values to 0. Cheap to compute. Prevents saturation unlike sigmoid.
  • Max Pooling: 2×2 window, stride 2 → each output = maximum of 4 inputs. Halves spatial dimensions. Provides local translation invariance.
  • Stacking layers: early layers detect edges/corners; middle layers detect textures; deep layers detect shapes/objects (hierarchical features).
b)What makes CNNs particularly well-suited for image processing compared to standard fully-connected neural networks?
Answer

Local connectivity: each neuron connects to only a small local region (not all pixels) — exploits spatial locality. Weight sharing: the same filter applies everywhere → drastically fewer parameters (e.g., 1M-pixel FC = billions of params; CNN = tractable). Translation invariance: a feature is detected regardless of where in the image it appears (via pooling). FC networks do not exploit spatial structure and would overfit on image data.

Detail
  • An FC network on a 256×256 RGB image with 1000 hidden units = 256×256×3×1000 ≈ 200M parameters in the first layer alone. Intractable.
  • CNN with 32 filters of 3×3 on same image = 32×3×3×3 = 864 parameters in first layer.
  • Translation invariance: if a cat's ear is detected in the top-left, the same filter detects it in the bottom-right. After max-pooling, the exact position doesn't matter.
  • Hierarchical feature learning: features learned at each layer build on the previous, automatically discovering useful representations without manual feature engineering.
c)Explain transfer learning in the context of CNNs. What are its advantages?
Answer

Pre-train a CNN on a large dataset (e.g., ImageNet: 1.2M images, 1000 classes). The network learns general visual features (edges → textures → shapes). Transfer learned weights to a new task with limited data. Feature extraction: freeze conv layers, retrain only the classifier head (when tasks are similar). Fine-tuning: retrain all layers with small LR (when tasks differ). Advantages: requires far less data, trains much faster, typically achieves better accuracy than training from scratch.

Detail
  • Why it works: early CNN layers learn universal features (edge detectors, colour blobs) that transfer across domains — a network trained on photos can detect features in medical images.
  • Feature extraction: fastest approach. Only the final few layers are replaced (e.g., 1000-class head → 5-class head). Good when training data is small.
  • Fine-tuning: unfreeze all layers and continue training with a very small LR (to avoid destroying pre-learned features). Better when new task differs significantly from ImageNet.
  • Models like VGG, ResNet, Inception are standard pre-trained backbones available in libraries like TorchVision and Keras Applications.
Q9 Recurrent Neural Networks: memory, LSTM architecture, and the vanishing gradient problem. ▼
RNNs / LSTMLecture 10
a)How does an RNN maintain memory across time steps? What problem does it have with long-term dependencies?
Answer

An RNN maintains a hidden state hₜ updated each step: hₜ = tanh(W_h·hₜ₋₁ + W_x·xₜ). This acts as "memory" — carrying past information forward. Problem: during Backpropagation Through Time (BPTT), gradients are multiplied by the weight matrix at every step. If weights <1, gradients shrink exponentially → early time steps get ~0 gradient → the model cannot learn long-range dependencies.

Detail
  • Example: in "The cat that ate the fish was full" — connecting "cat" to "was" across many words requires gradients to flow back several steps without vanishing.
  • RNNs also suffer from exploding gradients in some cases (less common) — addressed with gradient clipping.
  • RNNs are inherently sequential → cannot parallelise across time steps → slow for long sequences.
b)Describe the LSTM architecture. How does it solve the vanishing gradient problem?
Answer

LSTM adds a cell state (Cₜ) as a long-term memory highway. Three gates control it: Forget gate (fₜ): what to erase from Cₜ₋₁. Input gate (iₜ): what new info to add. Output gate (oₜ): what to output. Cell update: Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ — via addition, not multiplication → gradient flows unchanged → no vanishing. GRU is simpler: merges forget+input into one update gate.

Detail
  • Forget gate: fₜ = σ(Wf·[hₜ₋₁, xₜ]). Near 0 = forget past info; near 1 = keep it.
  • Input gate: iₜ = σ(Wi·[hₜ₋₁, xₜ]); candidate: C̃ₜ = tanh(Wc·[hₜ₋₁, xₜ]).
  • Output gate: hₜ = oₜ ⊙ tanh(Cₜ) — filtered cell state becomes new hidden state.
  • The cell state uses element-wise multiplication by gates (near 1 = identity path) rather than weight matrix multiplication → gradient highway through time.
  • GRU: update gate z merges forget+input; reset gate r controls how much past state to use. No separate cell state. Fewer parameters; often comparable performance to LSTM.
c)What is the vanishing gradient problem in RNNs, and what are two approaches to address it?
Answer

During BPTT, gradients are multiplied by the weight matrix W at every time step. If the spectral radius of W <1, gradients decay exponentially → model can't learn long-range patterns. Solution 1 — LSTM/GRU: gating creates an additive gradient path through cell state, bypassing multiplicative decay. Solution 2 — Attention mechanisms: allows the model to directly attend to any input position, completely removing the sequential gradient bottleneck (basis of Transformers).

Detail
  • Gradient clipping addresses the exploding gradient (complementary issue) but does not fix vanishing.
  • LSTM/GRU: the cell state passes through time with only sigmoid/tanh element-wise operations (not full weight matrix multiplication) → gradients don't compound.
  • Attention: instead of squeezing all information into a fixed hidden state, attention scores allow the decoder to look directly at any encoder hidden state. Eliminates the need for gradients to travel through sequential time steps at all.
Q10 Generative AI: LLM architecture, Transformer encoder/decoder, and ChatGPT vs original Transformer. ▼
Generative AILecture 11
a)What is Generative AI? Explain the high-level architecture of Large Language Models (LLMs) such as ChatGPT.
Answer

Generative AI creates new content (text, images, audio) by learning data distributions — unlike discriminative models that just classify. LLM pipeline: Tokenise input → embed tokens (dense vectors) → add positional encoding (sine/cosine for word order) → stack of Transformer decoder layers (masked self-attention + FFN + residual + LayerNorm) → output probability distribution over vocabulary → autoregressive generation (one token at a time). Pre-trained on massive text corpora; fine-tuned for tasks.

Detail
  • Tokenisation: text split into sub-word tokens. Example: "I Love to code in python" → ["I","Love","to","code","in","python"].
  • Positional encoding: Transformer processes tokens in parallel (not sequentially), so positional information must be injected via sine/cosine encoding added to embeddings.
  • GPT-3: 96 decoder layers, 175B parameters. GPT-4: multimodal (text + image). GPT-1 (2018): 12 layers, 117M params.
  • Autoregressive generation: at each step, model outputs a probability over the full vocabulary. The most likely (or sampled) token is appended to the input; process repeats until end-of-sequence token.
b)What is the role of the encoder and decoder in the original Transformer architecture? (Vaswani et al., 2017)
Answer

Encoder: processes full input sequence bidirectionally (each token attends to all others). 6 identical layers: multi-head self-attention + position-wise FFN + residual + LayerNorm. Outputs contextualised representations. Used in BERT for understanding tasks. Decoder: generates output autoregressively. 6 layers with 3 sub-layers: (1) masked self-attention (can't see future tokens), (2) cross-attention to encoder output, (3) FFN. Scaled Dot-Product Attention: softmax(QKᵀ/√d_k)·V.

Detail
  • Encoder self-attention: Q, K, V all come from the same sequence. Each token attends to all others — bidirectional context.
  • Decoder masked self-attention: positions can only attend to earlier positions. "Masking" = filling future positions with −∞ before softmax → probability = 0 for future tokens. Ensures causality (autoregression).
  • Cross-attention (decoder's 2nd sub-layer): Q from decoder, K and V from encoder output. Allows decoder to "look at" the full input when generating each output token.
  • Multi-head attention: runs h=8 parallel attention functions with different learned projections. Each head captures different types of dependencies. Outputs concatenated then projected.
  • d_model = 512. FFN inner dimension = 2048. Positional encoding: PE(pos,2i) = sin(pos/10000^(2i/d_model)).
c)How does ChatGPT's architecture differ from the original Transformer? What is RLHF and why is it used?
Answer

Original Transformer: encoder + decoder (for seq-to-seq tasks). ChatGPT (GPT): decoder-only — no encoder, no cross-attention, only masked self-attention → strictly unidirectional (left-to-right). Generates one token at a time. RLHF (Reinforcement Learning from Human Feedback): (1) supervised fine-tuning on expert conversations, (2) train reward model from human preference rankings, (3) RL with PPO to maximise reward. Used because raw LLMs predict tokens but may generate harmful or unhelpful content — RLHF aligns behaviour to human values.

Detail
  • GPT = decoder-only: token at position t attends only to tokens 1..t-1. No encoder means no cross-attention layer in the stack.
  • GPT-3: 96 Transformer decoder layers, 175B parameters. This scale enables impressive few-shot learning.
  • RLHF Step 1 — Supervised Fine-Tuning (SFT): human trainers write example (prompt, good response) pairs. Model fine-tuned to produce similar responses.
  • RLHF Step 2 — Reward Model: model generates several responses; trainers rank them. A separate reward model learns to predict which response humans prefer.
  • RLHF Step 3 — PPO: treat language model as policy; reward model provides scalar feedback. PPO updates the policy to maximise reward while staying close to the SFT model (KL penalty).
  • Why RLHF: instruction-following, harmless responses, factual accuracy — these cannot be captured by next-token prediction alone.

Q1 — General ML

RELATED How did the Nutrition Advisor project face the challenge of "non-representative training data"? ▼

The Open Food Facts dataset (299,299 rows) is a crowd-sourced database, meaning product entries depend on what contributors choose to upload. This introduces sampling bias: European (especially French) products are over-represented, while products from other regions are sparse. As a result, the trained classifier may generalise less well to nutritional products from under-represented markets.

Additionally, many products had missing nutritional values (handled via imputation or removal). The project used stratified splits (60%/20%/20%) to ensure Nutri-Score class proportions were preserved across train/validation/test sets, mitigating within-split imbalance.

RELATED The Random Forest achieved test accuracy of 90.2% — how did you ensure this wasn't overfitting? ▼

Three safeguards were used:

  • Stratified hold-out test set (59,860 samples, 20%) was never used during training or hyperparameter search, ensuring an unbiased final evaluation.
  • GridSearchCV with 3-fold cross-validation on the training set for hyperparameter tuning. The model was selected based on cross-validation performance, not test set performance.
  • Out-of-bag (OOB) error is implicitly available in Random Forest (each tree is evaluated on the ~36.8% samples not in its bootstrap sample), providing a built-in overfitting check.

The final test macro-F1 of 0.884 is consistent with the validation performance, confirming the model generalises rather than overfits.

RELATED What would have gone wrong if GridSearchCV had tuned hyperparameters directly on the 20% test set? ▼

The 59,860-sample test set would have effectively become part of training. Each hyperparameter combination tried would push the model to perform well on that specific split. After many trials, the selected model would be over-fitted to the test set's particular quirks — the test macro-F1 of 0.884 would be inflated and would not predict real-world performance. Re-evaluating on a truly unseen dataset would likely show a lower score. Using cross-validation on the training set, as the project did, prevents this by keeping the test set completely blinded.

Q2 — Performance Metrics

RELATED Why was macro-F1 chosen as the primary metric instead of accuracy for the Nutri-Score classifier? ▼

The Open Food Facts dataset has an imbalanced class distribution across Nutri-Score grades A–E. If class D had far more samples than class A, a model that predicts "D" for everything could achieve high accuracy yet be useless for identifying healthy products (grade A).

Macro-F1 computes the F1-score separately for each class and averages them with equal weight, regardless of class size. This forces the model to perform well across all five grades, including minority classes. The Random Forest achieved macro-F1 0.884 vs. Logistic Regression's 0.620 — a difference that would have been partially masked by accuracy alone.

RELATED Class B had a lower F1 than other grades in your Random Forest. What does that imply about the confusion matrix? ▼

A lower F1 for class B (F1 ≈ 0.775 vs overall macro 0.884) implies the model struggles to clearly separate B from its neighbouring classes — most likely A (slightly healthier) and C (slightly less healthy). In the confusion matrix this would manifest as:

  • FP for B: some A or C products predicted as B (reducing precision).
  • FN for B: some true-B products predicted as A or C (reducing recall).

This is expected because the nutritional boundaries between adjacent Nutri-Score grades are subtle. Products near the A/B or B/C threshold share very similar feature values (energy, fat, sugar, salt), making them inherently hard to classify.

Q3 — Gradient Descent & Regularisation

RELATED Why did Logistic Regression (macro-F1 0.620) perform so much worse than Random Forest (0.884)? ▼

Logistic Regression optimised with gradient descent assumes a linear decision boundary between Nutri-Score classes. The relationship between nutritional features (energy, fat, sugar, salt, fibre, protein, carbs) and Nutri-Score is highly non-linear — no simple linear combination of those features cleanly separates grades A–E.

This is a classic high-bias / underfitting scenario: the model is too simple for the data's complexity. Random Forest, being an ensemble of non-linear decision trees, can capture complex interactions (e.g., "high salt AND high saturated fat → grade D") that Logistic Regression cannot.

Adding regularisation (Ridge/Lasso) to Logistic Regression would help avoid overfitting but would not fix the fundamental linear boundary limitation — it would still underfit the non-linear Nutri-Score function.

RELATED How does the bias-variance tradeoff relate to tuning Random Forest's max_depth hyperparameter? ▼

A very deep tree (max_depth = None, unconstrained) has low bias (captures complex patterns) but high variance (overfits individual bootstrap samples). A shallow tree (small max_depth) has high bias but low variance.

The project's GridSearchCV searched over max_depth values (e.g., None vs 20) to find the sweet spot. The Random Forest's bagging mechanism already reduces much of the variance from deep trees, so in practice max_depth=None often works well — the ensemble averaging cancels out individual tree overfitting. This is one reason Random Forests are robust and a good default choice.

Q4 — Ensemble Learning

RELATED Explain why Random Forest was a better choice than a single Decision Tree for the Nutri-Score classifier. ▼

A single decision tree is a high-variance model: a small change in training data can produce a very different tree. On a dataset of 299,299 products, a single tree would likely overfit to noise in the nutritional values.

Random Forest addresses this by:

  • Training hundreds of trees on different bootstrap samples — each sees a different subset of products.
  • Using a random subset of features at each split — decorrelates the trees so their errors don't all go in the same direction.
  • Majority voting across all trees — individual errors cancel out, leaving the robust signal.

Result: the Random Forest achieved macro-F1 0.884 vs a single tree which would be expected to be significantly lower and more sensitive to the specific train/test split.

RELATED Feature importance from your Random Forest showed salt (0.2296) as most important. How does this relate to ensemble learning principles? ▼

Random Forest feature importance is computed as the mean decrease in impurity (Gini impurity) across all trees and all splits where a feature is used. A high importance for salt (0.2296) means that when any tree splits on salt, it produces the greatest reduction in node impurity — i.e., salt most strongly separates Nutri-Score classes.

This aligns with the Nutri-Score calculation: salt is a "bad" nutrient with high weight in the official Nutri-Score scoring algorithm. The full importance ranking: salt (0.2296) > saturated fat (0.1456) > sugars (0.1336) > energy (0.1228) > proteins (0.1057) > fat (0.1040) > carbs (0.0922) > fibre (0.0665).

Q5 — Dimensionality Reduction

RELATED Your recommender uses cosine similarity on 8-dimensional nutrition vectors. What would PCA do if applied first? ▼

The recommender's feature space is already 8-dimensional (energy, fat, sat. fat, carbs, sugars, fibre, proteins, salt) — low enough that dimensionality reduction is not necessary for computational reasons.

Applying PCA first would create new orthogonal components combining all 8 features. The first few principal components would capture the most variance, potentially noise-reducing the feature space. However:

  • It would reduce interpretability — you can no longer say "similar in salt and sugar."
  • PCA maximises variance, not class separability, so it might not improve Nutri-Score-relevant similarity.
  • With only 8 features and 299,299 samples, the curse of dimensionality is not a concern; PCA would add complexity without clear benefit.
RELATED How does the feature importance from Random Forest relate to feature selection concepts from the dimensionality reduction lecture? ▼

Random Forest feature importance is a form of embedded feature selection — feature relevance is determined during model training rather than in a separate preprocessing step. It is analogous to wrapper-method feature selection in principle but performed implicitly within the ensemble.

In the Nutrition Advisor project, the importance ranking (salt > saturated fat > sugars > energy > proteins > fat > carbs > fibre) tells us which features most reduce classification impurity. If we applied forward feature selection, we would likely arrive at a similar ordering — adding salt first, then saturated fat, etc. This is feature selection: choosing a subset of the original 8 nutritional features, not transforming them.

Q6 — ANNs

RELATED If you replaced the Random Forest classifier with a neural network, what activation function would you use and why? ▼

For the hidden layers: ReLU (or variants like ELU/Leaky ReLU). The 8 nutritional features → Nutri-Score classification is a tabular data problem. ReLU's non-saturation in the positive region avoids vanishing gradients, enabling efficient training even with several hidden layers.

For the output layer: Softmax, since this is 5-class classification (Nutri-Score A–E). Softmax converts the final logits to class probabilities summing to 1, from which we extract the predicted grade.

However, for tabular data with 8 features, a simple MLP would likely not outperform Random Forest significantly, as tree-based ensembles are generally superior on low-dimensional tabular data. The ANN would require careful tuning of layers, neurons, dropout, and learning rate.

RELATED The ml_analyse_product function returns class probabilities (predict_proba). How are these used in the serving size discount? ▼

The function ml_analyse_product(product) calls best_pipe.predict_proba(feat)[0] to get the probability distribution across Nutri-Score classes A–E.

From these probabilities, a health score is defined as: health_score = P(A) + P(B) — the combined probability of the product being grade A or B (healthy grades).

The ML-driven serving discount formula is: discount = 0.5 + 0.5 × health_score, ranging from 0.5 (worst case, P(A)+P(B)=0) to 1.0 (best case, certain grade A/B). This is then applied as: recommended_g = round(raw_serving_grams × discount).

This elegant use of class probabilities (not just the argmax prediction) allows a continuous, graded recommendation rather than a binary allow/deny decision.

Q7 — Training Deep NNs

RELATED How does GridSearchCV with 3-fold cross-validation relate to early stopping and preventing overfitting? ▼

Both cross-validation and early stopping serve the same goal — choosing model complexity/parameters that generalise, not just train well.

  • GridSearchCV with 3-fold CV: for each hyperparameter combination, the training set is split into 3 folds. The model trains on 2 folds and evaluates on 1. This is repeated 3 times. The combination with best average validation score is selected. This prevents choosing parameters that just overfit the specific training partition.
  • Early stopping: in iterative training (neural nets, gradient boosting), training is halted when validation loss starts increasing — the point where overfitting begins. Analogous to 3-fold CV choosing the optimal "stopping point" in hyperparameter space.

In the Nutrition Advisor project, 3-fold CV effectively acted as the "validation monitor" for the Random Forest hyperparameter search (n_estimators, max_depth, etc.).

Q8 — CNNs

RELATED Your project uses barcode scanning (image input). How does that relate to CNN capabilities? ▼

The scan_barcode() function in the notebook captures an image from the camera and decodes the barcode. This is a computer vision step but uses classical barcode decoding (pattern matching on vertical bars), not a CNN.

If a CNN were involved, it would be for cases where the barcode is damaged or partially obscured — a CNN trained to detect and localise barcodes (similar to object detection) could handle such cases. Transfer learning from a pre-trained object-detection CNN (e.g., YOLO) could be fine-tuned to identify barcodes in challenging conditions.

Once the barcode number is decoded, fetch_product_from_api(barcode) retrieves nutritional data from the Open Food Facts API, and then the ML pipeline takes over with tabular feature data — entirely separate from CNN processing.

RELATED Could transfer learning be applied to improve the Nutri-Score classifier? Explain. ▼

Transfer learning in the CNN sense (reusing visual features from ImageNet) doesn't directly apply to tabular nutritional data.

However, an analogous transfer learning concept does apply: a Random Forest or neural network pre-trained on a larger nutritional database (e.g., USDA FoodData Central) could be fine-tuned on the Open Food Facts data. The model would have already learned which nutritional features correlate with healthiness.

For a neural network specifically: pre-train on a large nutritional dataset, then freeze early layers (general feature detection) and fine-tune the final classification layers on Open Food Facts — directly analogous to CNN transfer learning. This would be particularly useful if Open Food Facts data were smaller or more noisy.

Q9 — RNNs

RELATED Your project has no user history — if you had temporal scan data, how would LSTM help? ▼

The current system treats each product scan independently. An LSTM-based extension could model sequences of scanned products over days/weeks:

  • Input sequence: [product_scan_1, product_scan_2, ..., product_scan_t] (each with nutritional features + time of day + meal context).
  • LSTM hidden state carries forward accumulated nutritional intake history.
  • Output: personalised next-product recommendation or nutrient gap prediction.

Standard RNN would fail here because eating patterns have long-term dependencies ("user always skips breakfast on Mondays") that span many time steps — exactly the long-term dependency problem RNNs can't handle. LSTM's cell state would retain this information across hundreds of scan events. GRU would also work with fewer parameters.

Q10 — Generative AI

RELATED Your system generates weekly diet plans using rule-based logic. How could a Transformer-based LLM improve this? ▼

The current generate_diet_plan(user) uses deterministic rules: BMR → TDEE → daily targets → product selection by cosine similarity. It cannot adapt to user preferences, cultural food habits, or conversational refinement.

A Transformer-based LLM integration could:

  • Natural language interface: "I'm vegetarian and dislike spinach — give me a high-protein plan." The LLM interprets this and adjusts product selection accordingly.
  • Plan explanation: generate human-readable explanations for why certain products were recommended ("We chose this product because its salt content is 0.3g/100g, helping you stay within your sodium limit").
  • Contextual adaptation: the decoder architecture generates a personalised plan token-by-token, attending to the user's full health profile and history via the attention mechanism.
RELATED The discount formula (0.5 + 0.5 × health_score) is hand-designed. How could RLHF be applied to learn it? ▼

The current discount formula is a human-designed heuristic. RLHF could replace it with a learned serving-size policy:

  • Step 1 — Supervised fine-tuning: collect examples from nutrition experts rating recommended serving sizes as appropriate/too large/too small. Train a model to mimic expert serving suggestions.
  • Step 2 — Reward model: show users pairs of serving recommendations; record which they prefer (or ask nutritionists to rank). Train a reward model to predict preference scores.
  • Step 3 — RL (PPO): treat the serving-size recommender as a policy. Use the reward model as feedback. The policy learns to adjust serving sizes to maximise both nutritional appropriateness and user satisfaction — potentially discovering a better function than the linear 0.5 + 0.5 × health_score.
