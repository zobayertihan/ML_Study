Q1 General Machine Learning: challenges, overfitting, and hyperparameter tuning pitfalls. ▼
General MLLecture 1–3

a) Main challenges in machine learning

  • Insufficient training data – most algorithms need thousands of examples to work well.
  • Non-representative training data – sampling bias means the training set doesn't reflect the real distribution (e.g., only data from one region).
  • Poor-quality data – outliers, errors, and noise cause the model to learn wrong patterns.
  • Irrelevant features – "garbage in, garbage out"; feature engineering (selection/extraction) is critical.
  • Overfitting – model memorises training data, fails on new instances (high variance).
  • Underfitting – model too simple to capture underlying structure (high bias).
  • No Free Lunch theorem – no single algorithm always wins; choice depends on the problem.

b) Model performs perfectly on training but badly on new data — overfitting

This is called overfitting. The model has memorised training examples rather than learning the general relationship.

Solutions:

  • Regularisation (Ridge/Lasso/Elastic Net) — penalise large weights.
  • Gather more training data.
  • Reduce noise in the dataset.
  • Simplify the model (fewer parameters, shallower tree, fewer layers).
  • Cross-validation to detect overfitting early.
  • Dropout (neural networks) — randomly deactivate neurons during training.
  • Early stopping — halt training when validation loss starts increasing.

c) What can go wrong when tuning hyperparameters on the test set?

Repeatedly optimising hyperparameters against the test set causes data leakage: the test set effectively becomes part of training. The model is then tuned to the specific quirks of that test set rather than the true data distribution. Reported performance will be over-optimistic and will not reflect real-world generalisation.

Solution: Use a separate validation set (or k-fold cross-validation) for hyperparameter search; keep the test set completely held-out as a final unbiased evaluation.

Q2 Performance measurements: regression metrics, confusion matrix, and AUC-ROC curve. ▼
Performance MetricsLecture 4

a) Regression performance metrics

  • MAE (Mean Absolute Error) = average of |ŷᵢ − yᵢ|. Robust to outliers. Easiest to interpret: average prediction error in the same units as the target.
  • MSE (Mean Squared Error) = average of (ŷᵢ − yᵢ)². Penalises large errors more. Differentiable, so useful for training.
  • RMSE (Root MSE) = √MSE. Same units as target. Most commonly reported. Lower is better.
  • R² (coefficient of determination) = 1 − SS_res/SS_tot. Proportion of variance explained. R²=1 is perfect; R²=0 is no better than predicting the mean; R²<0 is worse than the mean.

b) Confusion matrix and derived metrics

A confusion matrix shows the counts of TP (true positives), TN (true negatives), FP (false positives), FN (false negatives).

  • Accuracy = (TP+TN)/(TP+TN+FP+FN). Misleading for imbalanced classes.
  • Precision = TP/(TP+FP). "Of all positive predictions, how many are truly positive?" — quality.
  • Recall (Sensitivity) = TP/(TP+FN). "Of all actual positives, how many did we catch?" — coverage.
  • F1-score = 2×(P×R)/(P+R). Harmonic mean; useful when both precision and recall matter equally.
  • Macro-F1 = average F1 per class, treating each class equally regardless of size. Best for imbalanced multi-class problems.

c) ROC curve and AUC

The ROC curve plots True Positive Rate (TPR = Recall) on the y-axis vs. False Positive Rate (FPR = FP/(FP+TN)) on the x-axis, sweeping across all possible decision thresholds.

  • AUC = 0.5: random classifier (diagonal line — no discrimination).
  • AUC = 1.0: perfect classifier.
  • Perfect classifier shape: the curve goes straight up from (0,0) to (0,1) then horizontally to (1,1) — an L-shape hugging the top-left corner.

AUC is threshold-invariant — it evaluates the model across all possible cut-offs, making it useful for comparing classifiers independent of any specific threshold.

Q3 Gradient Descent variants, bias-variance tradeoff, and regularisation (Ridge, Lasso, Elastic Net). ▼
Gradient DescentLecture 3

a) Gradient Descent and its variants

Gradient Descent minimises a cost function J(θ) by iteratively updating parameters in the direction of steepest descent: θ = θ − α · ∇J(θ), where α is the learning rate.

  • Batch GD: uses the entire training set to compute the gradient at each step. Stable, guaranteed convergence for convex functions. Very slow for large datasets.
  • Stochastic GD (SGD): uses one randomly chosen training instance per step. Fast updates; noisy convergence; can escape shallow local minima but bounces around the minimum.
  • Mini-batch GD: uses small random subsets (e.g., 32–256 samples). Best of both worlds — can exploit GPU parallelism, smoother convergence than SGD, faster than batch GD. Most widely used in practice (deep learning).

b) Bias-Variance tradeoff

  • Bias: error from overly simplistic assumptions. High bias → model underfits (misses patterns in both train and test data).
  • Variance: sensitivity to small fluctuations in training data. High variance → model overfits (great on train, poor on test).
  • Formula: Total error = Bias² + Variance + Irreducible noise.
  • Simple models (linear regression, shallow tree): high bias, low variance.
  • Complex models (deep trees, large neural nets): low bias, high variance.
  • Tradeoff: decreasing bias (making model more complex) usually increases variance, and vice versa. Goal: find the sweet spot that minimises total error on unseen data.

c) Ridge vs Lasso vs Elastic Net

  • Ridge (L2): adds λ · Σwᵢ² to loss. Shrinks all weights toward zero but never to exactly zero. Handles multicollinearity. Use when all features may be relevant and you need weight shrinkage.
  • Lasso (L1): adds λ · Σ|wᵢ| to loss. Drives some weights to exactly zero → automatic feature selection. Produces sparse models. Use when you suspect only a few features truly matter.
  • Elastic Net: combines L1 + L2. Balances feature selection and weight shrinkage. Handles correlated features better than pure Lasso (which picks one arbitrarily). Preferred when there are many correlated features or when Lasso is too unstable.
Q4 Ensemble Learning: bagging, boosting, base learner choice, and Random Forest. ▼
Ensemble LearningLecture 5

a) Bagging (Bootstrap Aggregating)

Introduced by Breiman (1996). Combines multiple weak learners trained independently in parallel.

  • Bootstrap sampling: draw N samples from training set with replacement. Each sample retains ~63.2% unique instances; ~36.8% are out-of-bag (OOB) and can be used for validation.
  • Train L independent base learners on each bootstrap sample.
  • Combine predictions: majority voting for classification, averaging for regression.
  • Why it works: diverse learners trained on different subsets → errors are decorrelated → lower variance in ensemble vs. any single model.
  • Random Forest = bagging of decision trees + random feature subsetting at each split (further decorrelates trees).

b) Bagging vs Boosting — and when to prefer bagging

AspectBaggingBoosting
TrainingParallel (independent)Sequential (each corrects previous)
FocusReduce varianceReduce bias
Base learnersHigh-variance, low-bias (deep trees)Weak, high-bias (stumps/shallow trees)
Noise sensitivityRobustCan overfit noisy data

Prefer bagging when: data is noisy, parallelisation is needed, base models already have high variance, or robustness is prioritised.

c) How base learner choice affects performance

  • For bagging: use high-variance, low-bias learners (e.g., deep unpruned decision trees). Bagging reduces their variance. Shallow trees in bagging = underpowered (not enough variance to reduce).
  • For boosting: use weak, high-bias learners (e.g., decision stumps). Boosting sequentially corrects their bias. Using complex base learners in boosting risks rapid overfitting.
  • Key principle: base learner type must be coherent with the aggregation strategy's goal (variance-reduction vs. bias-reduction).
Q5 Dimensionality Reduction: PCA, LDA vs PCA, feature selection vs feature extraction. ▼
Dimensionality ReductionLecture 6

a) PCA — role of eigenvectors and eigenvalues

PCA finds directions of maximum variance in the data and projects it onto a lower-dimensional subspace.

Steps: (1) Centre the data (subtract mean). (2) Compute covariance matrix Σ. (3) Find eigenvectors and eigenvalues of Σ. (4) Sort by descending eigenvalue. (5) Project data onto the top-k eigenvectors.

  • Eigenvectors: the principal components — directions of maximum variance. They are mutually orthogonal (perpendicular).
  • Eigenvalues: scalar values indicating the amount of variance explained in each eigenvector direction. The first principal component has the largest eigenvalue (captures most variance).
  • The explained variance ratio (eigenvalue / sum of all eigenvalues) tells you how many components to keep (e.g., keep 95% of variance).

b) PCA vs LDA — when to prefer LDA

  • PCA: unsupervised. Maximises total variance in data. Ignores class labels. May choose directions that don't separate classes. Up to min(N, D) components.
  • LDA: supervised (discriminative). Maximises ratio of between-class variance to within-class variance. Explicitly uses class labels for separability. At most (C−1) components (C = number of classes).
  • Example: In face recognition, Fisherfaces (LDA-based) outperform Eigenfaces (PCA-based) under varying illumination, because LDA learns class-specific discriminating features rather than just the highest-variance directions.
  • Prefer LDA when: you have class labels, classification is the goal, and you want the projection to maximise class separability.

c) Feature selection vs Feature extraction

  • Feature selection: chooses a subset of original features. Original features remain unchanged. Methods: forward/backward search, filter (correlation, mutual information), wrapper (RFE). Original features remain interpretable. Example: selecting "salt" and "saturated fat" from nutritional features.
  • Feature extraction: transforms the original feature space into a new (usually lower-dimensional) space. New features are combinations of originals. Methods: PCA (linear), LDA, autoencoders (non-linear). New features may not be directly interpretable. Example: PCA on nutritional data creates components that are linear combinations of all nutrients.
Q6 Artificial Neural Networks: perceptron, activation functions, and preventing overfitting. ▼
ANNsLecture 7

a) The perceptron model and its limitations

The perceptron (Rosenblatt, 1957) is the simplest neural network unit. It computes: z = Σwᵢxᵢ + b, then applies a step function: output = 1 if z ≥ 0, else 0.

Limitations:

  • Can only solve linearly separable problems (cannot learn XOR).
  • The step function is non-differentiable → cannot use gradient descent.
  • No hidden layers → cannot learn complex, non-linear mappings.
  • Solution: Multi-Layer Perceptron (MLP) with differentiable non-linear activation functions and backpropagation.

b) Activation functions — role and examples

Role: Introduce non-linearity. Without them, any stack of linear layers reduces to a single linear transformation — a deep network would be equivalent to a shallow one.

  • Sigmoid: σ(x) = 1/(1+e⁻ˣ), range [0,1]. Squashes output to probability range. Problem: vanishing gradients (derivative ≤ 0.25), slow training in deep nets.
  • Tanh: range [−1,1]. Zero-centred (better than sigmoid). Still suffers from vanishing gradients in very deep nets.
  • ReLU: f(x) = max(0,x). No saturation for positive inputs → gradient = 1 there. Fast, sparse activation. Default choice for hidden layers. Problem: "dying ReLU" (neuron stuck at 0 if input always negative).
  • Leaky ReLU / ELU: fix dying ReLU by allowing small negative activations. Less prone to vanishing gradients.
  • Softmax: converts logits to probabilities summing to 1. Used in output layer for multi-class classification.

c) Preventing overfitting in neural networks

  • Dropout: randomly deactivates a fraction (e.g., 50%) of neurons each training step. Forces the network to learn redundant, robust representations. Not applied at inference.
  • L1/L2 regularisation: add penalty term to loss function. L2 (weight decay) shrinks all weights; L1 drives some to zero (sparse weights).
  • Early stopping: monitor validation loss; stop training when it stops decreasing. Prevents memorisation of training set.
  • Batch normalisation: normalises layer inputs per mini-batch, reducing internal covariate shift. Also has an implicit regularisation effect.
  • Data augmentation: artificially expand training data (flip, rotate, crop, add noise). More data = better generalisation.
Q7 Training Deep Neural Networks: vanishing/exploding gradients, learning rate scheduling, optimisers. ▼
Deep Learning TrainingLecture 8

a) Vanishing and exploding gradients — causes, effects, and remedies

Vanishing gradient: during backpropagation, gradients shrink exponentially as they travel from output to early layers. With sigmoid, the derivative is always ≤ 0.25; multiplied through N layers, it approaches zero. Lower layers receive negligible gradient signal and stop learning. Described in detail by Glorot & Bengio (2010).

Exploding gradient: gradients grow exponentially, causing massive, unstable weight updates and NaN values. Mostly occurs in RNNs.

Remedies:

  • Use ReLU or Leaky ReLU: derivative = 1 for positive inputs → no saturation → gradient flows freely.
  • Proper weight initialisation: Xavier/Glorot for sigmoid/tanh; He initialisation for ReLU.
  • Batch Normalisation: keeps activations in the healthy (non-saturating) region of sigmoid/tanh.
  • Gradient clipping: rescale gradients when their norm exceeds a threshold — prevents explosion without changing direction.
  • LSTM/GRU: gating mechanisms create linear pathways for gradients through time in RNNs.
  • Residual connections: skip connections provide a direct gradient highway bypassing layers.

b) Learning rate — why it matters and two scheduling strategies

LR too high → diverges or oscillates. LR too low → slow convergence, may get stuck.

  • Step decay: reduce LR by a factor (e.g., ×0.1) every N epochs. Simple and effective. Allows fast initial learning then fine-grained tuning.
  • Cosine annealing: LR follows a cosine curve from max to near zero. Smooth decay without abrupt drops. Often combined with warm restarts to escape local minima.

c) Adam vs RMSProp optimisers

  • RMSProp (Hinton, 2012): maintains an exponential moving average of squared gradients per parameter. Scales each parameter's LR inversely with root of that average. Adapts to each parameter individually. Good for non-stationary objectives (e.g., RNNs). Avoids AdaGrad's monotonically shrinking LR.
  • Adam (Kingma & Ba, 2014): combines momentum (1st moment = running mean of gradients) + RMSProp (2nd moment = running mean of squared gradients). Bias-corrected estimates. Formula: θ = θ − α · m̂/√(v̂+ε). Generally faster convergence than RMSProp. The default choice for most deep learning tasks. More robust but has more hyperparameters.
Q8 Convolutional Neural Networks: architecture, advantages over FC networks, and transfer learning. ▼
CNNsLecture 9

a) CNN architecture for image classification

  • Input layer: raw pixel values, e.g., 224×224×3 (H×W×channels).
  • Convolutional layer: applies learnable filters (kernels) across the image via sliding window. Each filter produces a feature map detecting a specific pattern (edges, textures). Multiple filters → stack of feature maps. Uses weight sharing — same filter applied everywhere.
  • Activation (ReLU): applied element-wise after convolution. Sets negative values to zero. Introduces non-linearity. Produces a stack of images with no negative values.
  • Pooling layer (Max Pooling): reduces spatial dimensions by taking the maximum value in each local window (e.g., 2×2, stride 2). Provides translation invariance. Reduces computational load and parameter count.
  • Stacked layers: Conv → ReLU → Pool blocks repeated multiple times. Early layers: low-level features (edges). Later layers: high-level features (shapes, objects).
  • Fully connected layers: flattened feature maps → dense connections → combine features for classification.
  • Output (Softmax): converts logits to class probabilities summing to 1.

b) Why CNNs outperform standard fully-connected NNs for images

  • Local connectivity: each neuron connects to only a small local region (receptive field), exploiting spatial locality of features.
  • Weight sharing: the same filter is applied across all positions → massively fewer parameters. A 1M-pixel image with 1000 classes in a FC net would require billions of parameters; CNNs are tractable.
  • Translation invariance: after pooling, a feature is detected regardless of its location in the image. CNNs recognise X's and O's even when translated, scaled, or slightly rotated.
  • Hierarchical feature learning: stacking layers learns progressively abstract representations automatically from raw pixels.

c) Transfer learning

Pre-train a CNN on a large dataset (e.g., ImageNet: 1.2M images, 1000 classes). The network learns general visual features (edges → textures → shapes → objects). Transfer these learned weights to a new task with limited data.

  • Feature extraction: freeze all convolutional layers; replace and retrain only the final classifier head. Fast; works when new task is similar to pre-training task.
  • Fine-tuning: unfreeze some or all layers and retrain with a small learning rate. Better when new domain differs significantly.
  • Benefits: requires far less labelled data, converges much faster, typically achieves superior accuracy compared to training from scratch.
Q9 Recurrent Neural Networks: memory, LSTM architecture, and the vanishing gradient problem. ▼
RNNs / LSTMLecture 10

a) How RNNs maintain memory and their limitation

An RNN maintains a hidden state h_t updated at each time step: h_t = tanh(W_h · h_{t-1} + W_x · x_t). This hidden state carries information forward in time, acting as the model's "memory." The output at each step depends on both the current input and the previous hidden state.

Problem with long-term dependencies: During backpropagation through time (BPTT), gradients are multiplied by the weight matrix at every time step. If weights < 1, gradients shrink exponentially → early time steps receive almost zero gradient → the model cannot learn relationships between events separated by many steps (long-term dependencies).

b) LSTM architecture — how it solves vanishing gradients

LSTM introduces a separate cell state (C_t) that serves as a long-term memory highway alongside the hidden state.

  • Forget gate (f_t = σ(W_f · [h_{t-1}, x_t])): decides what to erase from cell state. Near 0 = forget, near 1 = keep.
  • Input gate (i_t = σ(W_i · [h_{t-1}, x_t])): decides what new information to write. Combined with candidate: C̃_t = tanh(W_c · [h_{t-1}, x_t]).
  • Cell state update: C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t — update via addition, not multiplication.
  • Output gate (o_t = σ(W_o · [h_{t-1}, x_t])): h_t = o_t ⊙ tanh(C_t).

Why it solves vanishing gradients: The cell state propagates through time via addition rather than multiplication, creating a linear "gradient highway." Gradients can flow back unchanged through many time steps, allowing the model to retain relevant information over long sequences.

GRU (Gated Recurrent Unit): simpler variant that merges forget and input gates into a single update gate and merges cell state and hidden state. Fewer parameters, often similar performance to LSTM.

c) Two approaches to address vanishing gradients in RNNs

  • LSTM/GRU: gating mechanisms create additive gradient pathways. The cell state highway allows gradients to flow backward without exponential decay.
  • Attention mechanisms: allow the model to directly attend to any position in the input sequence at each decoding step, completely bypassing the sequential gradient bottleneck. Forms the basis of the Transformer architecture.
  • Gradient clipping: cap gradient norm to prevent exploding gradients (standard practice in RNN training, though it doesn't fix vanishing).
Q10 Generative AI: LLM architecture, Transformer encoder/decoder, and ChatGPT vs original Transformer. ▼
Generative AILecture 11

a) What is Generative AI? High-level architecture of LLMs

Generative AI: a branch of AI that creates new, original content (text, images, audio, video) by learning the distribution of training data. Unlike discriminative models that classify, generative models aim to produce new samples consistent with that distribution.

High-level LLM pipeline:

  • Tokenisation: input text → tokens (sub-word units). Example: "I Love to code in python" → ["I","Love","to","code","in","python"].
  • Embedding: tokens → dense vector representations (Word2Vec, GloVe, or learnt embeddings).
  • Positional encoding: sine/cosine functions added to embeddings to inject word-order information (since Transformer processes in parallel, not sequentially).
  • Transformer decoder layers (stacked; GPT-3 has 96): masked multi-head self-attention → feed-forward network → residual connections + layer normalisation.
  • Output projection: hidden states → probability distribution over vocabulary. Autoregressive generation: sample one token at a time, append to input, repeat until end token.
  • Pre-training on massive text corpus (hundreds of billions of tokens) + fine-tuning for specific tasks.

b) Role of encoder and decoder in the Transformer

Encoder (Vaswani et al., "Attention Is All You Need", 2017):

  • Processes the full input sequence bidirectionally (each token attends to all others simultaneously).
  • 6 identical layers, each containing: multi-head self-attention + position-wise feed-forward network + residual connection + layer normalisation. Output dimension: d_model = 512.
  • Produces contextualised hidden representations for each input token.
  • Used in models like BERT for understanding tasks (classification, QA).

Decoder:

  • Generates output autoregressively, one token at a time.
  • 6 layers, each with 3 sub-layers: (1) masked self-attention (cannot attend to future positions — ensures causality); (2) cross-attention over encoder output (aligns decoder to input context); (3) position-wise FFN.
  • The masking implements "autoregression" — the decoder only sees what it has already generated.

Attention mechanism: maps Query (Q), Key (K), Value (V) vectors. Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V. Multi-head attention runs h attention functions in parallel to capture different types of dependencies.

c) ChatGPT vs original Transformer — and RLHF

  • Original Transformer: encoder + decoder. Used for seq-to-seq tasks (e.g., machine translation). Decoder uses cross-attention to attend to encoder output.
  • ChatGPT (GPT series): decoder-only. No encoder; no cross-attention. Uses only masked self-attention → strictly unidirectional (left to right). Each token attends only to previous tokens. Generates text one token at a time using the conversation history as context. GPT-3: 96 Transformer decoder layers, 175B parameters.
  • RLHF (Reinforcement Learning from Human Feedback): trains the model to be helpful, harmless, and honest:
    1. Supervised fine-tuning: human AI trainers write example conversations; model is fine-tuned on these.
    2. Reward model: trainers rank different model responses by quality; a reward model is trained to predict human preference scores.
    3. RL with PPO: the language model (policy) is trained using Proximal Policy Optimisation to maximise the reward model's scores.
  • Why RLHF? A raw language model predicts the next likely token but may generate harmful, biased, or unhelpful content. RLHF aligns the model's behaviour to match human values and conversational goals.

Q1 — General ML

RELATED How did the Nutrition Advisor project face the challenge of "non-representative training data"? ▼

The Open Food Facts dataset (299,299 rows) is a crowd-sourced database, meaning product entries depend on what contributors choose to upload. This introduces sampling bias: European (especially French) products are over-represented, while products from other regions are sparse. As a result, the trained classifier may generalise less well to nutritional products from under-represented markets.

Additionally, many products had missing nutritional values (handled via imputation or removal). The project used stratified splits (60%/20%/20%) to ensure Nutri-Score class proportions were preserved across train/validation/test sets, mitigating within-split imbalance.

RELATED The Random Forest achieved test accuracy of 90.2% — how did you ensure this wasn't overfitting? ▼

Three safeguards were used:

  • Stratified hold-out test set (59,860 samples, 20%) was never used during training or hyperparameter search, ensuring an unbiased final evaluation.
  • GridSearchCV with 3-fold cross-validation on the training set for hyperparameter tuning. The model was selected based on cross-validation performance, not test set performance.
  • Out-of-bag (OOB) error is implicitly available in Random Forest (each tree is evaluated on the ~36.8% samples not in its bootstrap sample), providing a built-in overfitting check.

The final test macro-F1 of 0.884 is consistent with the validation performance, confirming the model generalises rather than overfits.

RELATED What would have gone wrong if GridSearchCV had tuned hyperparameters directly on the 20% test set? ▼

The 59,860-sample test set would have effectively become part of training. Each hyperparameter combination tried would push the model to perform well on that specific split. After many trials, the selected model would be over-fitted to the test set's particular quirks — the test macro-F1 of 0.884 would be inflated and would not predict real-world performance. Re-evaluating on a truly unseen dataset would likely show a lower score. Using cross-validation on the training set, as the project did, prevents this by keeping the test set completely blinded.

Q2 — Performance Metrics

RELATED Why was macro-F1 chosen as the primary metric instead of accuracy for the Nutri-Score classifier? ▼

The Open Food Facts dataset has an imbalanced class distribution across Nutri-Score grades A–E. If class D had far more samples than class A, a model that predicts "D" for everything could achieve high accuracy yet be useless for identifying healthy products (grade A).

Macro-F1 computes the F1-score separately for each class and averages them with equal weight, regardless of class size. This forces the model to perform well across all five grades, including minority classes. The Random Forest achieved macro-F1 0.884 vs. Logistic Regression's 0.620 — a difference that would have been partially masked by accuracy alone.

RELATED Class B had a lower F1 than other grades in your Random Forest. What does that imply about the confusion matrix? ▼

A lower F1 for class B (F1 ≈ 0.775 vs overall macro 0.884) implies the model struggles to clearly separate B from its neighbouring classes — most likely A (slightly healthier) and C (slightly less healthy). In the confusion matrix this would manifest as:

  • FP for B: some A or C products predicted as B (reducing precision).
  • FN for B: some true-B products predicted as A or C (reducing recall).

This is expected because the nutritional boundaries between adjacent Nutri-Score grades are subtle. Products near the A/B or B/C threshold share very similar feature values (energy, fat, sugar, salt), making them inherently hard to classify.

Q3 — Gradient Descent & Regularisation

RELATED Why did Logistic Regression (macro-F1 0.620) perform so much worse than Random Forest (0.884)? ▼

Logistic Regression optimised with gradient descent assumes a linear decision boundary between Nutri-Score classes. The relationship between nutritional features (energy, fat, sugar, salt, fibre, protein, carbs) and Nutri-Score is highly non-linear — no simple linear combination of those features cleanly separates grades A–E.

This is a classic high-bias / underfitting scenario: the model is too simple for the data's complexity. Random Forest, being an ensemble of non-linear decision trees, can capture complex interactions (e.g., "high salt AND high saturated fat → grade D") that Logistic Regression cannot.

Adding regularisation (Ridge/Lasso) to Logistic Regression would help avoid overfitting but would not fix the fundamental linear boundary limitation — it would still underfit the non-linear Nutri-Score function.

RELATED How does the bias-variance tradeoff relate to tuning Random Forest's max_depth hyperparameter? ▼

A very deep tree (max_depth = None, unconstrained) has low bias (captures complex patterns) but high variance (overfits individual bootstrap samples). A shallow tree (small max_depth) has high bias but low variance.

The project's GridSearchCV searched over max_depth values (e.g., None vs 20) to find the sweet spot. The Random Forest's bagging mechanism already reduces much of the variance from deep trees, so in practice max_depth=None often works well — the ensemble averaging cancels out individual tree overfitting. This is one reason Random Forests are robust and a good default choice.

Q4 — Ensemble Learning

RELATED Explain why Random Forest was a better choice than a single Decision Tree for the Nutri-Score classifier. ▼

A single decision tree is a high-variance model: a small change in training data can produce a very different tree. On a dataset of 299,299 products, a single tree would likely overfit to noise in the nutritional values.

Random Forest addresses this by:

  • Training hundreds of trees on different bootstrap samples — each sees a different subset of products.
  • Using a random subset of features at each split — decorrelates the trees so their errors don't all go in the same direction.
  • Majority voting across all trees — individual errors cancel out, leaving the robust signal.

Result: the Random Forest achieved macro-F1 0.884 vs a single tree which would be expected to be significantly lower and more sensitive to the specific train/test split.

RELATED Feature importance from your Random Forest showed salt (0.2296) as most important. How does this relate to ensemble learning principles? ▼

Random Forest feature importance is computed as the mean decrease in impurity (Gini impurity) across all trees and all splits where a feature is used. A high importance for salt (0.2296) means that when any tree splits on salt, it produces the greatest reduction in node impurity — i.e., salt most strongly separates Nutri-Score classes.

This aligns with the Nutri-Score calculation: salt is a "bad" nutrient with high weight in the official Nutri-Score scoring algorithm. The full importance ranking: salt (0.2296) > saturated fat (0.1456) > sugars (0.1336) > energy (0.1228) > proteins (0.1057) > fat (0.1040) > carbs (0.0922) > fibre (0.0665).

Q5 — Dimensionality Reduction

RELATED Your recommender uses cosine similarity on 8-dimensional nutrition vectors. What would PCA do if applied first? ▼

The recommender's feature space is already 8-dimensional (energy, fat, sat. fat, carbs, sugars, fibre, proteins, salt) — low enough that dimensionality reduction is not necessary for computational reasons.

Applying PCA first would create new orthogonal components combining all 8 features. The first few principal components would capture the most variance, potentially noise-reducing the feature space. However:

  • It would reduce interpretability — you can no longer say "similar in salt and sugar."
  • PCA maximises variance, not class separability, so it might not improve Nutri-Score-relevant similarity.
  • With only 8 features and 299,299 samples, the curse of dimensionality is not a concern; PCA would add complexity without clear benefit.
RELATED How does the feature importance from Random Forest relate to feature selection concepts from the dimensionality reduction lecture? ▼

Random Forest feature importance is a form of embedded feature selection — feature relevance is determined during model training rather than in a separate preprocessing step. It is analogous to wrapper-method feature selection in principle but performed implicitly within the ensemble.

In the Nutrition Advisor project, the importance ranking (salt > saturated fat > sugars > energy > proteins > fat > carbs > fibre) tells us which features most reduce classification impurity. If we applied forward feature selection, we would likely arrive at a similar ordering — adding salt first, then saturated fat, etc. This is feature selection: choosing a subset of the original 8 nutritional features, not transforming them.

Q6 — ANNs

RELATED If you replaced the Random Forest classifier with a neural network, what activation function would you use and why? ▼

For the hidden layers: ReLU (or variants like ELU/Leaky ReLU). The 8 nutritional features → Nutri-Score classification is a tabular data problem. ReLU's non-saturation in the positive region avoids vanishing gradients, enabling efficient training even with several hidden layers.

For the output layer: Softmax, since this is 5-class classification (Nutri-Score A–E). Softmax converts the final logits to class probabilities summing to 1, from which we extract the predicted grade.

However, for tabular data with 8 features, a simple MLP would likely not outperform Random Forest significantly, as tree-based ensembles are generally superior on low-dimensional tabular data. The ANN would require careful tuning of layers, neurons, dropout, and learning rate.

RELATED The ml_analyse_product function returns class probabilities (predict_proba). How are these used in the serving size discount? ▼

The function ml_analyse_product(product) calls best_pipe.predict_proba(feat)[0] to get the probability distribution across Nutri-Score classes A–E.

From these probabilities, a health score is defined as: health_score = P(A) + P(B) — the combined probability of the product being grade A or B (healthy grades).

The ML-driven serving discount formula is: discount = 0.5 + 0.5 × health_score, ranging from 0.5 (worst case, P(A)+P(B)=0) to 1.0 (best case, certain grade A/B). This is then applied as: recommended_g = round(raw_serving_grams × discount).

This elegant use of class probabilities (not just the argmax prediction) allows a continuous, graded recommendation rather than a binary allow/deny decision.

Q7 — Training Deep NNs

RELATED How does GridSearchCV with 3-fold cross-validation relate to early stopping and preventing overfitting? ▼

Both cross-validation and early stopping serve the same goal — choosing model complexity/parameters that generalise, not just train well.

  • GridSearchCV with 3-fold CV: for each hyperparameter combination, the training set is split into 3 folds. The model trains on 2 folds and evaluates on 1. This is repeated 3 times. The combination with best average validation score is selected. This prevents choosing parameters that just overfit the specific training partition.
  • Early stopping: in iterative training (neural nets, gradient boosting), training is halted when validation loss starts increasing — the point where overfitting begins. Analogous to 3-fold CV choosing the optimal "stopping point" in hyperparameter space.

In the Nutrition Advisor project, 3-fold CV effectively acted as the "validation monitor" for the Random Forest hyperparameter search (n_estimators, max_depth, etc.).

Q8 — CNNs

RELATED Your project uses barcode scanning (image input). How does that relate to CNN capabilities? ▼

The scan_barcode() function in the notebook captures an image from the camera and decodes the barcode. This is a computer vision step but uses classical barcode decoding (pattern matching on vertical bars), not a CNN.

If a CNN were involved, it would be for cases where the barcode is damaged or partially obscured — a CNN trained to detect and localise barcodes (similar to object detection) could handle such cases. Transfer learning from a pre-trained object-detection CNN (e.g., YOLO) could be fine-tuned to identify barcodes in challenging conditions.

Once the barcode number is decoded, fetch_product_from_api(barcode) retrieves nutritional data from the Open Food Facts API, and then the ML pipeline takes over with tabular feature data — entirely separate from CNN processing.

RELATED Could transfer learning be applied to improve the Nutri-Score classifier? Explain. ▼

Transfer learning in the CNN sense (reusing visual features from ImageNet) doesn't directly apply to tabular nutritional data.

However, an analogous transfer learning concept does apply: a Random Forest or neural network pre-trained on a larger nutritional database (e.g., USDA FoodData Central) could be fine-tuned on the Open Food Facts data. The model would have already learned which nutritional features correlate with healthiness.

For a neural network specifically: pre-train on a large nutritional dataset, then freeze early layers (general feature detection) and fine-tune the final classification layers on Open Food Facts — directly analogous to CNN transfer learning. This would be particularly useful if Open Food Facts data were smaller or more noisy.

Q9 — RNNs

RELATED Your project has no user history — if you had temporal scan data, how would LSTM help? ▼

The current system treats each product scan independently. An LSTM-based extension could model sequences of scanned products over days/weeks:

  • Input sequence: [product_scan_1, product_scan_2, ..., product_scan_t] (each with nutritional features + time of day + meal context).
  • LSTM hidden state carries forward accumulated nutritional intake history.
  • Output: personalised next-product recommendation or nutrient gap prediction.

Standard RNN would fail here because eating patterns have long-term dependencies ("user always skips breakfast on Mondays") that span many time steps — exactly the long-term dependency problem RNNs can't handle. LSTM's cell state would retain this information across hundreds of scan events. GRU would also work with fewer parameters.

Q10 — Generative AI

RELATED Your system generates weekly diet plans using rule-based logic. How could a Transformer-based LLM improve this? ▼

The current generate_diet_plan(user) uses deterministic rules: BMR → TDEE → daily targets → product selection by cosine similarity. It cannot adapt to user preferences, cultural food habits, or conversational refinement.

A Transformer-based LLM integration could:

  • Natural language interface: "I'm vegetarian and dislike spinach — give me a high-protein plan." The LLM interprets this and adjusts product selection accordingly.
  • Plan explanation: generate human-readable explanations for why certain products were recommended ("We chose this product because its salt content is 0.3g/100g, helping you stay within your sodium limit").
  • Contextual adaptation: the decoder architecture generates a personalised plan token-by-token, attending to the user's full health profile and history via the attention mechanism.
RELATED The discount formula (0.5 + 0.5 × health_score) is hand-designed. How could RLHF be applied to learn it? ▼

The current discount formula is a human-designed heuristic. RLHF could replace it with a learned serving-size policy:

  • Step 1 — Supervised fine-tuning: collect examples from nutrition experts rating recommended serving sizes as appropriate/too large/too small. Train a model to mimic expert serving suggestions.
  • Step 2 — Reward model: show users pairs of serving recommendations; record which they prefer (or ask nutritionists to rank). Train a reward model to predict preference scores.
  • Step 3 — RL (PPO): treat the serving-size recommender as a policy. Use the reward model as feedback. The policy learns to adjust serving sizes to maximise both nutritional appropriateness and user satisfaction — potentially discovering a better function than the linear 0.5 + 0.5 × health_score.
